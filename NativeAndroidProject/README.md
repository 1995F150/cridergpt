# Native Android Project Notes

This module contains the **actual Android-native implementation** (Compose screens, repositories, app shell) for CriderGPT.

## Scope after cleanup

- Kept: native app shell, screen routing, auth/chat/todo repositories, resources.
- Removed: autogenerated one-file-per-web-component placeholder scaffolds under `migrated/` that did not provide executable Android behavior.

## Why this cleanup

The previous scaffold created hundreds of placeholder Kotlin files that mirrored web TS/TSX names but did not advance Android feature parity. Removing these files keeps the Android module maintainable and focused on working code paths.

## Next incremental migration approach

When porting a web feature, add only:

1. A real Compose UI implementation.
2. Supporting ViewModel/repository/data models.
3. Navigation route(s) and tests where practical.

Avoid creating placeholder files for unimplemented features.


## Pricing and checkout alignment

- Native pricing now loads plans from Supabase `plan_configurations` (same source used by web flows).
- Native payment now invokes the `create-checkout` edge function with real `priceId`/`planName` values.
- Final subscription state is still applied by the existing Stripe webhook functions on Supabase.
